ImageWAM: Do World Action Models Really Need Video Generation, or Just Image Editing?

📄 arXiv
2606.19531 · arXiv · code

video 생성 기반 WAM이 미래 영상 전체를 합성하는 비용/오류를, pretrained image-editing 모델(OmniGen2 main, +FLUX.2/Ovis-U1)을 backbone으로 재활용해 대체. 추론 시 target frame을 픽셀로 디코딩하지 않고 image-editing denoising의 KV cache(edit-aware 중간표현)에 flow-matching Action Expert를 joint attention으로 conditioning → action chunk a_{t:t+H} 예측. video-WAM 대비 FLOPs ~1/6·latency ~1/4로 VLA baseline(π0/π0.5 등) 능가. manipulation을 'instruction-guided visual transformation'으로 보는 관점.

기여image-editing 모델을 instruction-conditioned visual backbone으로 repurpose하는 ImageWAM 프레임워크(video 생성 WAM의 대안) 제시; robot manipulation을 instruction-guided visual transformation으로 정식화하고 editing pretraining의 3속성(instruction-to-change alignment / easier·action-relevant proxy / compact inference) 식별; editing-aware representation이 standard visual·VLM backbone보다 action-relevant함을 비교로 실증.
강점
  • 효율: dense future-video 디코딩 제거 → FLOPs ~1/6 · latency ~1/4 (실시간 control 적합)
  • image editing = video보다 action-relevant·easier proxy: 현재→target의 instruction-consistent 변화에만 집중, 무관 temporal/appearance detail에 capacity 안 씀
  • 부정확 future-video가 action을 오도하는 문제 회피 (Fig 5: video baseline은 task-relevant 객체 주변을 distort → 실패)
  • 여러 image-editing backbone에 일반화: OmniGen2(main)·FLUX.2·Ovis-U1 (appendix)
  • RoboTwin 2.0·LIBERO·LIBERO-Plus·real-world에서 VLA baseline 능가
약점
  • pretrained image-editing 모델(OmniGen2 등) prior에 의존 — 그 품질·도메인이 성능을 좌우
  • 미래를 '단일 image-edit(한 장)'으로 표현 → multi-step temporal dynamics를 명시적으로 모델링하진 않음 (WAM 미래표현 중 가장 압축적)

Output-Level Regularization Eliminates the Seed Lottery in Single-GPU VLA Fine-Tuning

🎓 CoRL 2026 (submitted)
2606.13856 · arXiv

단일 GPU에서 VLA(VLA-JEPA)를 fine-tuning할 때 동일 코드·동일 데이터로도 random seed에 따라 한 run이 조용히 붕괴하는 'seed lottery'를, output-level regularization(VICReg/Dropout/낮춘 LR)으로 제거한 연구. 같은 코드 13회 중 12회는 91~94% 성공, 1회는 65.2%로 붕괴하는 현상을 진단·해결한다. Skoltech.

기여seed lottery 현상을 Jacobian null-space 분석으로 정량 특성화하고, regularizer를 output-level vs weight-level로 가르는 taxonomy를 제시한다. weight-level(L2/EWC)은 output collapse를 못 막는 반면 output-level(VICReg/Dropout/낮춘 LR)은 모든 catastrophic seed를 제거 — optimizer config 한 줄 수준의 실용 해법.
강점
  • 동일 setup 13회 반복의 통계적으로 견고한 실험 설계 — 분산 차이 F(12,11)=28.7, p<0.001로 검증
  • collapse를 success rate뿐 아니라 Jacobian null-space·effective rank 등 다각도로 진단해 'weight-level이 왜 무력한지'를 형식화
  • 약 30줄 PyTorch recipe(VICReg/Dropout/낮춘 LR) — single-GPU 저비용 환경에 즉시 적용·재현 용이
약점
  • 검증이 LIBERO 시뮬레이션 + VLA-JEPA 단일 backbone에 한정 — 일반화 근거 약함
  • L2/EWC·LowLR·Dropout 등 일부 method가 single-run/소수 seed(n≈3~5)로만 측정 → baseline(n=13)과의 분산 비교가 비대칭
  • 코드/체크포인트 미공개 (acceptance 이후 공개 예정)

  • LIBERO는 실제 실행 지연을 반영하지 못해 인위적 제약으로 real-time을 모사
  • 복잡한 시나리오용 AR action expert 아키텍처는 open question

VLA-JEPA

📄 arXiv
2602.10098 · arXiv · code

멀티뷰 RGB 장면 카메라와 언어 지시를 입력받아, 장면이 어떻게 변할지를 픽셀이 아닌 추상 latent(JEPA embedding) 공간에서 상상하면서 연속 end-effector action chunk를 출력하는 VLA 정책이다. VLM 백본(Qwen3-VL-2B)이 latent action token과 action token을 내면, 작은 predictor(latent world model)가 미래 latent state를 예측하고 flow-matching head(DiT-B)가 행동을 생성한다. leakage-free state prediction으로 appearance bias·카메라 motion·정보 누출에 강건한 조작 정책을 달성했다(LIBERO 97.2%, LIBERO-Plus 79.5%).

기여기존 latent-action VLA pretraining은 latent가 raw 픽셀 변화에 anchoring돼 appearance bias·nuisance motion·information leakage에 취약했는데, VLA-JEPA는 미래 frame을 V-JEPA2 target encoder로 인코딩해 지도 target으로만 쓰고 학습자는 현재 관측만 보게 하는 leakage-free state prediction으로 이를 설계 차원에서 회피한다. 다단계 latent-action 파이프라인 대신 JEPA pretraining → action-head fine-tuning의 단순한 2단계 레시피로 일반화·강건성을 끌어올린 점이 새롭다.
강점
  • leakage-free state prediction: 미래 frame을 지도 target으로만 사용하고 학습자는 현재만 보게 해 latent action이 미래를 베껴 치팅하는 information leakage를 구조적으로 차단
  • 픽셀이 아닌 JEPA latent 공간에서 dynamics를 예측해 카메라 motion·배경 변화 등 action-irrelevant한 디테일에 강건
  • human video(alignment loss만) + robot data(alignment+action joint)의 cross-domain co-training으로 행동 라벨 없는 대규모 영상도 world-model 학습에 활용
  • LIBERO 97.2%, LIBERO-Plus 79.5%, SimplerEnv(Google 65.2%/WidowX 57.3%)로 π0.5·OpenVLA-OFT·UniVLA·villa-X 등 강한 baseline 대비 일관된 우위 + 코드·가중치 공개
약점

  • 분류 경계가 모호: 미래 latent 예측은 pretraining 보조 목적이고 행동은 별도 flow-matching head라 ② WAM이되 ①(VLM+ActionExpert)로 읽을 여지도 있음(③/④는 분명히 아님)

  • 코드·base model·벤치 데이터(GM-100)·프로젝트 페이지를 모두 공개한 오픈소스 파운데이션 모델

Learning Latent Action World Models In The Wild (JEPA-Wild)

🔴 Meta (FAIR)📄 arXiv
2601.05230 · arXiv

행동 라벨이 전혀 없는 in-the-wild 인터넷 영상(YouTube-Temporal-1B, 약 20M 영상·10억 프레임)을 입력으로, V-JEPA 2 encoder가 만든 latent 공간에서 다음 frame latent를 예측하는 world model과, 연속 frame 사이의 latent action을 복원하는 IDM을 함께 학습한다. 출력은 예측된 next-frame latent이며, 작은 controller가 실 로봇 행동을 latent action으로 매핑해 현재 이미지+목표 이미지로부터 goal-conditioned 행동 sequence를 plan한다. 라벨 없이 학습했음에도 action-conditioned baseline과 유사한 planning 성능을 달성한다.

기여기존 latent-action world model이 로봇 시뮬·게임·조작 데이터에 국한되고 대개 vector quantization(VQ)으로 행동을 discretize했던 것과 달리, 풍부한 실세계 dynamics를 잡기 위해 VQ 대신 sparsity/noise 정보 병목으로 정규화한 연속(continuous-but-constrained) latent action 공간을 라벨 없이 IDM으로 복원한다. 이를 10억 프레임 규모의 uncurated 인터넷 영상으로 scale하고, 영상 간 환경 변화 transfer와 카메라 기준으로 localize된 행동 학습을 보이며, controller(universal interface)로 실 로봇 planning까지 연결한 점이 새롭다.
강점
  • 라벨 없는 uncurated 인터넷 영상(약 10억 프레임)으로 latent-action world model을 인터넷 규모로 scale
  • VQ(고정 codebook snap)가 무너뜨리는 풍부한 in-the-wild motion을, 연속이되 sparsity/noise 제약을 둔 latent action으로 포착 — VQ 대비 우위를 명시적으로 보임
  • 영상 간 환경 변화 transfer(예: 방에 사람이 들어오는 변화) 등 in-the-wild 특화 행동 학습을 입증
  • 가벼운 controller가 실 로봇 행동↔latent action을 매핑하는 universal interface 역할로, 라벨 없이 학습했음에도 action-conditioned baseline과 유사한 planning 성능 달성
약점
  • 공통 embodiment이 없어 학습된 latent action이 카메라 기준 공간에 localize되는 데 그침 — 일반화된 행동 의미를 직접 학습하지는 못함(저자 자인 limitation)
  • 정확한 planner(CEM/receding-horizon MPC)는 선행작 V-JEPA 2-AC에서의 추론이며 본 논문이 직접 명시한 것이 아님
  • 공식 코드·가중치 repo 미확인, 학습데이터(YT-Temporal-1B)도 GCS 버킷 셧다운으로 원본 프레임 접근 불가(메타데이터만, 유튜브 직접 수집 필요) — 재현성 제약
  • abstract·메타데이터에 정량 벤치마크 수치가 제시되지 않아 planning 성능이 'baseline과 유사' 수준의 정성 서술에 머무름; pixel-decoder loss가 정확히 L1인지 등 일부 세부는 미확인

InternVLA-A1

📄 arXiv
2601.02456 · arXiv · code

멀티뷰 장면 이미지(현재+최근 과거 frame) + 언어 지시(+로봇 proprioception q_t)를 입력받아, 하나의 Mixture-of-Transformers 안에서 미래 visual latent(foresight)를 상상하고 그 상상에 조건화된 다음 action chunk a_{t:t+k}를 함께 출력하는 범용 로봇 조작 정책이다. understanding·generation·action 3개 expert가 단일 masked self-attention을 공유해 의미 이해와 미래 상상이 동시에 행동을 안내한다. pi0.5 대비 dynamic 조작에서 +26.7%를 보고했다.

기여표준 MLLM 기반 VLA는 물리 모델이 없어 dynamic task에 약하고, 순수 영상예측 WM 정책은 시각적으로 brittle하고 의미 grounding이 약했다. InternVLA-A1은 의미 이해(understanding expert)와 미래 visual foresight 생성(generation expert), 행동 생성(action expert)을 별도 모듈 직렬 연결이 아니라 단일 unified masked self-attention을 공유하는 하나의 Mixture-of-Transformers로 융합하고, blockwise mask로 understand→imagine→act 일방향 흐름을 강제해 상상한 미래가 항상 행동을 조건화하도록 만든 점이 새롭다.
강점
  • 미래 상상(visual foresight)과 행동을 하나의 self-attention을 공유하는 통합 network에서 co-train해, 별도 영상생성기→정책 파이프라인의 누적 오류 없이 의미 추론과 미래 예측이 함께 행동을 안내한다
  • blockwise mask로 understand→imagine→act 일방향 흐름을 강제해 행동이 항상 foresight에 종속되게 구조화 — 상상 영상 오류가 행동을 직접 망치는 것을 완화
  • 실로봇+시뮬+egocentric human video 혼합 692M frame으로 사전학습해 sim-to-real gap을 줄이고, dynamic manipulation에서 pi0.5 대비 +26.7%로 표준 VLA의 약점인 동적 task를 크게 개선

🔴 Meta FAIR🎓 TMLR
2512.24497 · arXiv · code

로봇의 현재 카메라 이미지와 목표 이미지(+선택적 proprioception)를 입력받아, 픽셀이 아닌 frozen ViT(주로 DINOv2, photorealistic 환경엔 DINOv3 ViT-L/16) latent 공간에서 action-conditioned forward world model로 다양한 행동 sequence의 결과를 상상하고, 상상한 최종 latent가 목표 latent와 L2 거리상 가장 가까운 행동 sequence를 CEM/Nevergrad planner로 탐색해 출력한다. 학습은 multi-step rollout MSE loss 하나뿐이며, 픽셀 생성도 학습된 policy head도 언어 입력도 없다. 결과적으로 내비게이션·조작 양쪽에서 선행 로봇 WM인 DINO-WM과 V-JEPA-2-AC를 능가하는 latent-planning 레시피를 제시한다.

강점

  • frozen 자기지도 ViT encoder를 그대로 쓰고 학습은 multi-step rollout MSE 하나뿐이라 학습이 단순·저렴(픽셀 재구성·정책 학습 불필요)
  • 내비게이션과 조작 양쪽 벤치마크에서 선행 DINO-WM·V-JEPA-2-AC를 동시에 능가

약점

  • test-time에 CEM/Nevergrad 샘플링 최적화로 행동을 매번 탐색하므로 학습된 policy head 대비 추론 비용·지연이 큼

  • abstract/표에 정량 수치(성공률 등)가 명시되지 않아 검증 가능한 headline number 부재(공개 코드/표로 별도 확인 필요)

BAGEL (BAGEL-7B-MoT)

🔴 ByteDance Seed📄 arXiv
2505.14683 · arXiv · code

텍스트/이미지(프롬프트, 사진, 또는 interleaved 시퀀스)를 입력받아, 텍스트 답변 그리고 새로 생성/편집한 이미지(VQA·T2I·free-form 편집·next-frame/visual foresight)를 출력하는 통합(unified) decoder-only 멀티모달 모델이다. 단일 token stream 위에서 understanding/generation 두 expert가 매 layer self-attention을 공유하는 MoT(7B active / 14B total) 구조로, 텍스트는 autoregressive, 이미지는 VAE latent 공간의 Rectified Flow로 생성한다. 조 단위 interleaved text·image·video·web token으로 scale 했을 때 free-form 편집·미래-frame 예측·3D/multiview 조작·world navigation 같은 emerging "world-modeling" 능력이 나타남을 보였다(GenEval 0.82, MMBench 85.0 등 오픈소스 unified 모델 SOTA). 단, BAGEL 자체는 action head가 없어 로봇 행동은 출력하지 않으며, 로봇에서는 파생 VLA(BagelVLA)를 통해 subgoal/미래-frame 신호 공급원으로만 쓰인다.

기여기존 unified 멀티모달 모델은 생성 모듈을 VLM에 덧붙이는 방식이라 이해-생성 간 정보 병목(단일 shared expert 강제 또는 frozen generator)이 있었는데, BAGEL은 understanding/generation 두 expert가 매 layer에서 동일한 self-attention을 공유하는 Mixture-of-Transformer-Experts(MoT)로 병목을 제거하고, 이미지를 의미용(SigLIP2 ViT)과 픽셀용(FLUX VAE latent + Rectified Flow) 두 종류 token으로 이원화했다. 이를 조 단위 interleaved token으로 scale 하자 단순 챗·T2I를 넘어 free-form 편집·visual foresight·world navigation 같은 emerging 능력이 나타남을 처음으로 보였고, 이 future-frame 예측 능력이 로봇 스택의 world-model 절반(subgoal 공급)으로 재활용된다.
강점
  • understanding/generation 2-expert가 매 layer self-attention을 공유하는 MoT로 이해-생성 병목 제거; 이미지를 의미(SigLIP2)와 픽셀-latent(FLUX VAE)로 이원 토큰화
  • scale 시 free-form 편집·미래-frame 예측·3D/multiview 조작·world navigation 등 emerging world-modeling 능력 발현 (단순 T2I/챗을 넘어섬)
  • 오픈소스 unified 모델 대비 생성·이해·편집 전반 SOTA (GenEval 0.82 / 0.88 w. rewriter, MMBench 85.0, GEdit-Bench, IntelligentBench Self-CoT 55.3)
  • 모델 가중치·코드 Apache-2.0 공개(ByteDance-Seed/BAGEL-7B-MoT); future-frame 능력이 BagelVLA의 subgoal/visual-foresight 절반으로 재사용 가능
약점

π0 / π0.5 (Physical Intelligence)

🔴 Physical Intelligence🎓 RSS 2025
2410.24164 · arXiv · code

멀티뷰 RGB 이미지 + 자연어 지시 (+관절각 proprioception)를 입력받아 H=50 step의 연속 action chunk(최대 ~50 Hz)를 직접 출력하는 단일 범용 VLA(Vision-Language-Action) 정책이다. PaliGemma(~3B) VLM 백본에 flow-matching 기반 action expert(~300M)를 붙여 빨래 개기·식탁 정리 같은 정밀 고주파 실로봇 조작을 말로 지시할 수 있게 했다. π0.5는 행동 전에 고수준 텍스트 "subtask"를 먼저 출력하는 hierarchical 구조와 웹·객체탐지 co-training을 더해 처음 보는 집/환경으로의 일반화를 달성했다.

기여기존 정책(OpenVLA, Octo)이 행동을 discrete token으로 잘라 정밀 고주파 동작에 거칠고 학습 장면 근처에서만 동작했던 한계를, 사전학습된 VLM은 그대로 유지하고 별도 flow-matching action expert를 붙여 noise→연속 action을 흘려보내는 방식으로 해결했다. 두 expert를 shared self-attention(block-wise causal mask)으로만 연결해 VLM 지식을 보존하면서 정밀 연속 제어를 얻고, π0.5는 "먼저 말로 생각(subtask) → 행동" co-training으로 미지 환경 일반화까지 확장한 점이 새롭다.
강점
  • discrete token 대신 flow matching으로 H=50, ~50 Hz의 정밀·연속·multimodal action chunk를 생성 — 빨래 개기 같은 dexterous 고주파 동작 가능
  • MoT(Mixture of Transformers) 2-expert 설계 + block-wise causal mask로 사전학습 VLM 지식을 오염 없이 보존하면서 action expert만 별도 학습
  • ~10,000시간·7개 로봇·68 task의 cross-embodiment 데이터 + OXE/DROID 혼합으로 단일 범용 정책 학습
  • π0.5의 hierarchical subtask co-training으로 처음 보는 집/환경 일반화 (zero-shot에 가까운 새 장면 동작)
  • 코드·가중치가 openpi로 공개되어 재현·확장 용이
약점

SuSIE (Zero-Shot Robotic Manipulation with Image-Editing Diffusion)

📄 arXiv
2310.10639 · arXiv · code

로봇 카메라 RGB 한 장 + 텍스트 명령("냄비에 숟가락 넣기")을 입력하면, 사전학습 image-editing diffusion(InstructPix2Pix)이 ~20 step 뒤 장면이 어때야 하는지를 사실적인 subgoal RGB 이미지로 그려내고, 별도의 작은 goal-conditioned diffusion BC 정책(gc_ddpm_bc)이 현재 이미지와 그 subgoal 이미지를 받아 action chunk(horizon=4)를 출력한다. 이 imagine-then-act 루프를 연쇄해 긴 task를 짧은 시각 목표들로 분해·완수하며, CALVIN(zero-shot ABC→D) SOTA와 실세계 Bridge-style task에서 RT-2-X 능가를 달성했다.

기여미래-상태 추론을 인터넷 사전학습 image generator(InstructPix2Pix)에 떠넘겨, 시스템이 넓은 시각/의미 일반화를 물려받아 새 객체·지시를 zero-shot으로 처리하면서도 low-level 정책은 단순·language-agnostic하게 유지한 것이 선행 대비 새로움이다. 예측 대상이 latent code가 아니라 눈에 보이는 RGB subgoal 이미지이고, 따로 학습된 WM(diffusion subgoal generator)과 IDM 역할 정책을 그 "이미지 한 장"으로만 연결하는 hierarchical 분해를 제시했다.
강점
  • 인터넷 규모 사전학습 image-editing diffusion의 시각/의미 일반화를 그대로 물려받아 새 객체·지시에 대한 zero-shot 처리 능력 확보
  • high-level(미래 관측 예측)과 low-level(행동 복원)을 분리해 low-level 정책을 단순·language-agnostic(BridgeData V2의 미수정 gc_ddpm_bc)으로 유지 — 모듈 재사용성 높음
  • 예측이 latent가 아닌 사람이 해석 가능한 RGB subgoal 이미지라 디버깅/검증 용이
  • CALVIN(zero-shot ABC→D) SOTA, 실세계 Bridge-style task에서 RT-2-X 능가
약점
  • WM과 IDM 정책이 jointly-trained가 아니라 따로 학습·순차 실행되어, 두 모델을 잇는 통로가 subgoal 이미지 한 장뿐 — high-level이 비현실적/도달 불가 subgoal을 그리면 그대로 행동을 망침(후속 GHIL-Glue가 subgoal 필터링으로 보완)
  • subgoal look-ahead ~20 step은 확실하나 정확한 replan 주기는 설정 의존적(CALVIN vs BridgeData/실세계)이라 고정된 설계값이 아님
  • arXiv 추상에 구체적 수치(성공률/벤치 점수)가 명시되지 않아 정량적 비교는 본문 표 확인 필요
  • 픽셀 공간 subgoal 생성(diffusion ~20 step denoise)이라 latent 기반 대비 high-level 추론 비용이 크고 추론 지연이 있을 수 있음
